6.3.4 Iterador

O elemento de composição Iterador permite que se execute uma série de comandos para todos as configurações de uma mesma classe ou sobre uma lista de valores. Este elemento é representado graficamente por um retângulo com uma indicação em preto no canto superior esquerdo, como ilustrado na figura 6.20.

O iterador pode ser configurado através do painel lateral ao selecioná-lo. O painel lateral de configuração do iterador é ilustrado na figura 6.19. O campo Modo é utilizado para indicar se a iteração é sobre os elementos de uma classe ou de uma lista. O campo Tipo é utilizado para definir o tipo dos elementos a serem iterados. No caso da iteração sobre todas as configurações de uma classe, esse tipo indica a classe a ser iterada. Já no caso de iteração sobre uma lista, a lista deve ser de elementos deste tipo. O campo Lista é utilizado para indicar a variável com a lista a ser iterada caso o modo seja Lista. O campo Variável é utilizado para indicar o nome da variável que deve ser utilizada para armazenar o elemento corrente da iteração. O campo Índice é utilizado para indicar o nome da variável que deve ser utilizara para armazenar o índice numérico do elemento corrente da interação. No caso de iteração sobre os elementos de uma classe, o campo Classe é utilizado para especificar a classe que deve ser iterada. Ao final do painel é possível definir uma descrição do elemento. Uma descrição em branco é substituída por uma descrição padrão.


Figura 6.19: Painel de configuração do iterador.

A figura 6.20 exemplifica o uso do elemento de composição Iteração sobre Equipamentos. O usuário deve criar uma Ligação com descrição I (ramo de Iteração) com origem no objeto Iterador e com o destino no primeiro elemento de composição do laço de iteração. Caso haja uma sequência do fluxo após o fim da iteração, o usuário deve criar uma Ligação com descrição F (fim) com origem no Iterador e destino no elemento de composição que dá sequência ao fluxo.


Figura 6.20: Exemplo de iteração sobre elementos.

A seguir são apresentadas quatro importantes notas sobre a utilização do objeto de composição Iteração sobre Equipamentos:

  • Os elementos de composição a partir da Ligação I (ou seja, que serão iterados) ocorrem em sequência. No exemplo da figura 6.20, o comando Avisar fechamento só ocorre após a execução do comando Fechar tanque tq.
  • A sequência do fluxo (a partir da Ligação F) só ocorre ao final da iteração. No exemplo da figura 6.20, o comando Tanques fechados só ocorre após o comando de avisar que o último tanque fechou (Avisar fechamento).
  • A variável de iteração só é disponível no ramo do fluxo iniciado a partir da Ligação I e o seu valor nunca é alterado (é constante), mesmo que haja alguma atribuição à variável de iteração.
  • O ramo do fluxo iniciado a partir da Ligação I deve ser um ramo de fluxo independente, ou seja, não pode ter ligação com o resto do fluxo. O exemplo da figura 6.21 mostra um exemplo de um fluxo errado em que o ramo de iteração se encontra com o ramo principal do fluxo.


Figura 6.21: Exemplo de erro na iteração sobre elementos.
