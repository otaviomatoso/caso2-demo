Capítulo 5
Configurações de Plantas

A edição de uma configuração de planta tem como objetivo definir os elementos do ambiente de automação a serem manipulados no projeto de aplicação do MPA. Esses elementos podem ser tanto equipamentos físicos, como válvulas e sensores; quanto elementos virtuais, como campos de uma tela de operação do sistema supervisório.

Uma configuração de planta consiste na definição de um conjunto de pontos de controle e equipamentos, que são chamados genericamente de instâncias. Cada instância apresenta um conjunto de atributos que definem suas características assim como seus relacionamentos com outras instâncias.

Um ponto de controle é uma instância associada a um elemento no sistema supervisório através de um identificador único denominado tag. Por sua vez, esse elemento no sistema supervisório está associado a uma variável de processo da planta, como um sensor ou um atuador. O MPA tem quatro classes de ponto de controle pré-definidas, uma para cada tipo de valor que um desses pontos pode assumir. As classes de ponto do MPA são: Pontos Lógicos, Pontos Inteiros, Pontos Reais e Pontos Textuais. Cada uma dessas classes possui um conjunto de atributos que podem ser definidos para uma instância daquela classe. Em particular, Pontos Inteiros e Pontos Reais podem definir os atributos fator e offset que são utilizados pelo MPA para ajustar os valores que são recebidos e enviados para o supervisório.

Um equipamento é um elemento abstrato que tipicamente representa um agregado de pontos de controle interrelacionados. Diferente das classes de pontos, as classes de equipamentos devem ser definidas em uma das bibliotecas de modelos carregada no projeto de aplicação. Cada atributo de uma classe de equipamento pode ser definido com valores de um único tipo. Entretanto, esse valor pode ser de um tipo básico, uma instância de classe ou uma lista.

Um equipamento de uma certa classe que tenha como atributo uma classe de ponto, ou outra classe de equipamento, precisa fazer referências a instâncias de outras classes. Essas referências têm que ser válidas para garantir que as operações sobre esses equipamentos não resultem em erros. Sendo assim, os pontos ou equipamentos referidos devem existir no momento que a aplicação é exportada em arquivo ou enviada para o executor (veja mais detalhes sobre exportação da aplicação na seção 3.2).

O editor de plantas facilita a manutenção dessas referências e realiza a verificação dos valores definidos para os atributos de pontos e equipamentos. Em particular, há uma configuração do editor de plantas para avisar durante a configuração de um equipamento que o ponto referido não foi configurado para planta (Menu Editar, Preferências... na seção 3.2.2). A partir do aviso o usuário escolhe como proceder: criando o ponto, ignorando o aviso ou desfazendo a configuração que gerou o aviso. Outro recurso é a visualização de uma configuração, com seus relacionamentos, por meio de grafos de ligação para acompanhar o desenvolvimento do seu projeto de planta.
