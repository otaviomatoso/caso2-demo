3.2.4 Menu Executor

Contém operações para controlar um servidor remoto de execução do MPA. Essas operações são divididas em dois grupos: o primeiro são operações sobre o servidor; e o segundo são operações sobre a aplicação MPA carregada no servidor.

Item Conectar.../Desconectar Conecta ou desconecta a IDE de um servidor de execução. Para se conectar a um servidor é necessário informar o nome da máquina do servidor, a senha de acesso e o número porta de conexão do processo naquela máquina. Isso é feito através de um diálogo que é exibido quando o comando é executado, como ilustrado na figura 3.3. Todas essas informações são configuradas no servidor de execução ao ser iniciado, como apresentado na seção 11.1.


Figura 3.3: Diálogo de conexão com servidor de execução.

Uma vez conectado, o comando muda para Desconectar, que permite encerrar o canal de comunicação com o servidor atual. Isso implica que as mensagens de acompanhamento não serão mais recebidas. Ao se desconectar do servidor de execução, toda a comunicação com o mesmo é encerrada, de forma que as mensagens de acompanhamento ou acompanhamento visual da execução da aplicação não serão automaticamente descontinuadas.

Item Encerrar... Inicia o processo de encerramento do servidor de execução ao qual a IDE esteja conectado. Neste caso, todos os diagramas são imediatamente interrompidos, a comunicação com a IDE é desconectada, e o processo do servidor de execução termina. É importante salientar que todos os serviços fornecidos pelo servidor de execução serão encerrados. Além disso, a IDE não fornece mecanismos para disparar o servidor de execução novamente.

Item Desbloquear... Esse comando fica habilitado sempre que a execução dos diagramas atinge um ponto de parada. Nesse caso, o comando permite prosseguir com a execução dos diagramas após o ponto de parada atingido. É importante salientar que outros usuários podem estar inspecionando a execução do diagrama nesse ponto de parada. Nesse caso, se a execução continuar o valor das variáveis sendo inspecionadas podem ser alteradas pela execução normal do diagrama fazendo com que o estado sendo inspecionado seja perdido.

Item Enviar Aplicação Exporta a aplicação atual através da conexão com o servidor de execução para que possa ser executada. Todas as definições da aplicação que não forem desabilitadas são incluídas no envio.

Quando definições de módulo são enviadas ao servidor, todas as classes e funções previamente carregadas no servidor são removidas. Portanto, para que o servidor aceite uma nova pré-configuração é necessário que não haja nenhuma configuração de planta ou diagramas presente no servidor, uma vez que elas podem fazer referência a alguma classe ou função da pré-configuração sendo substituída.

Não é possível substituir diagramas em execução ou suspensos, assim como enviar novas configurações de planta durante a execução de diagramas que façam referência às classes dessas configurações sendo enviadas.

Após o envio, os diagramas da aplicação ficam imediatamente disponíveis para execução no servidor, como é apresentado no capítulo 8.

Item Apagar Aplicação... Remove a aplicação carregada no servidor de execução ao qual a IDE esteja conectada. Só é permitido apagar a aplicação do servidor se não houver nenhum diagrama em execução. É importante salientar que todas as informações de configuração de planta e diagramas serão perdidas e, portanto, não é possível recuperá-las posteriormente. Entretanto, é possível reenviar ao servidor a mesma aplicação, bastando carregar um projeto com as definições originais de planta, diagramas e demais arquivos que compunham o projeto da aplicação.

Item Depurar Aplicação... Ativa o modo de depuração da aplicação, que permite um acompanhamento visual da execução dos diagramas em execução no servidor, assim como outros recursos para permitir compreender o funcionamento da aplicação e identificar possíveis inadequações com o intuito do desejado. A seção 6.6 apresenta os recursos do modo de depuração visual de aplicações MPA.

Item Remover Bloqueios... Remove quaisquer pontos de parada atualmente inseridos na aplicação carregada no servidor. Esse comando é tipicamente utilizado para limpar eventuais pontos de parada esquecidos por um usuário durante uma sessão de depuração da aplicação.

É importante salientar que outros usuários podem estar inspecionando utilizando esses pontos de parada para interromper a execução do diagrama em pontos estratégicos permitindo a inspeção de seu estado. Nesse caso, esse comando fará com que a definição dos pontos de parada sejam perdidas.
