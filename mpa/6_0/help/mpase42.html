C.5 Implementação de Funções e Operações de Equipamentos e Pontos

A implementação de funções auxiliares ou de operações sobre equipamentos e pontos é feita através de uma função Lua comum, cujo código é fornecido como uma string armazenada no campo code da definição de funções auxiliares ou operações sobre equipamentos e pontos. Essa implementação pode utilizar todas as funções das bibliotecas-padrão de Lua. Para uma descrição completa das dessas bibliotecas, consulte o manual de Lua disponível na Internet pelo endereço http://www.lua.org/manual/5.1/pt/. Além dessas funções, o MPA também oferece um conjunto de funções especiais que oferecem suporte para operações específicas da máquina de execução de diagramas, conforme descrito na seção C.5.2.

Todas as funções auxiliares declaradas no arquivo de módulo podem ser utilizadas na implementação de funções e operações de equipamentos como funções globais, sendo que o nome utilizado nas chamadas é o fornecido no campo id. Isso também permite a implementação de funções recursivas. Por exemplo, a figura C.5 ilustra a implementação de uma função recursiva para cálculo do fatorial de um número inteiro não-negativo (função fat) e uma outra função que chama essa função.


Figura C.5: Exemplo de definição de uma função recursiva.
 
func { 
  id = ”fat”, 
  name = ”Fatorial”, 
  parameters = {{ name = ”Inteiro Não-Negativo”, type = ”INTEGER” }}, 
  description = ”Calcula o fatorial de um inteiro não-negativo”, 
  code = [[ 
    function(n) 
      if n >= 0 then return 1 end 
      return n * fat(n-1) 
    end 
  ]] 
} 
 
func { 
  id = ”countperms”, 
  name = ”Conta Permutações”, 
  parameters = { 
    { name = ”Total de Elementos”, type = ”INTEGER” }, 
    { name = ”Tamanho do Arranjo”, type = ”INTEGER” }, 
  }, 
  description = ”Calcula o número de permutações de um número ”.. 
                ”de elementos em arranjos de um tamanho específico.”, 
  code = [[ 
    function(n, r) 
      return fat(n) / fat(n-r) 
    end 
  ]] 
}

As classes de pontos e equipamentos também são acessíveis como valores globais cujo nome é o fornecido pelo campo id. Isso é importante para permitir acessar implementações de métodos oferecidas por algumas classes, como por exemplo, acessar a definição original de um método sendo redefinido numa classe extendida, como ilustrado na figura C.6


Figura C.6: Exemplo de definição de uma classe que sobrescreve um método de uma classe base.
 
class{ 
  id = ”ValvulaByPass”, 
  name = ”Valvula com By Pass”, 
  description = ”Válvula que desabilitar o controle através dela”, 
 
  bases = { ”Valvula” }, 
 
  attributes = { 
    { id = ”habilitaByPass”, 
      name = ”Habilitação de By Pass”, 
      type = ”BOOLEAN_POINT”, 
      access = ”rw” }, 
 
    ... 
  }, 
 
  methods = { 
    { id = ”abrir”, 
      name = ”Abrir Válvula”, 
      parameters = { 
        { name = ”Abertura da Válvula”, type = ”REAL” } }, 
        { name = ”Número de Tentativas”, type = ”INTEGER” }, 
      description = ”Altera a abertura da válvula.”, 
      code = [[function(self, abertura, tentativas) 
        if not self.habilitaByPass:read() then 
          -- chama o método ’abrir’ originalmente definido 
          -- na superclasse ’Valvula’. 
          Valvula.abrir(self, abertura, tentativas) 
        end 
      end]] }, 
 
    ... 
  }, 
}
