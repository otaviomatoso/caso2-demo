Apêndice D
Servidor de Execução para OpenVMS

Este apêndice apresenta os passos para execução do servidor do MPA no sistema operacional OpenVMS. O servidor de execução do MPA (ou seja, o executor de diagramas) para o ambiente OpenVMS segue o mesmo padrão de configuração descrito neste manual, que também se aplica aos ambientes Windows e Unix.

O arquivo de distribuição (e.g. MPA_6_0_OPENVMS_8_3_VXL_5_0.ZIP) deve ser baixado para um diretório na máquina servidora (a que executa o OpenVMS), no tipo de transmissão imagem (type image, obtido com o comando BINARY do programa FTP), e seu conteúdo deve ser recuperado com o programa unzip, ou similar, para o diretório onde o servidor de execução do MPA será instalado. O pacote contém os seguintes arquivos:

MPA-MANUAL.PDF
este manual.
MPA-RUN.EXE
arquivo binário da versão VMS do executor MPA que possui a ponte de acesso à base de dados VXL.
VXLLUA.EXE
arquivo binário com console iterpretador lua com pacote vxl, cujo método open retorna a ponte de acesso à base de dados VXL.
RUN.COM
exemplo de script de comando para início do executor com passagem de parâmetros.
SERVICE.COM
exemplo de script de comando para início do executor como serviço na fila de processo “BATCH” do VMS.
START.COM
exemplo de script de comando para início de uma aplicação MPA.
STOP.COM
exemplo de script de comando para término de uma aplicação MPA.

Para o carregamento e execução do servidor, devem ser respeitadas algumas características do sistema operacional OpenVMS e de sua interface de comandos interativos (o DCL, Digital Command Languange) cujo sinal de pronto – ou prompt – é o $ por padrão.

O comando para executar um programa no DCL é o run. Para executar o servidor do MPA, é necessário o privilégio SYSLCK (system lock). O privilégio pode ser obtido com o seguinte comando1 (o usuário deve ter a autorização necessária)

$  set process/privilege=syslck

Para executar o servidor numa sessão interativa, pode-se usar

$  run MPA-RUN.EXE

Porém, como o comando run não permite passar parâmetros para o servidor do MPA, deve-se criar um nome simbólico (ou seja, uma variável de ambiente) com o comando:

$  MPA_RUN = "$ " + F$PARSE("MPA-RUN.EXE", "[direxec]",,,)

onde direxec é o diretório onde está o arquivo MPA-RUN.EXE.

Após isso, é possível iniciar o excutor do MPA passando-lhe parâmetros.

Por exemplo, o comando

$  MPA_RUN --host 10.10.228.253 --port 9091

carregará o executor, que aguardará comandos de uma IDE do MPA na interface de rede com o endereço IP 10.10.228.253, através da porta 9091 (em vez da porta 9090, que é a default).

Por comodidade, pode-se criar um arquivo de comandos de nome, por exemplo, RUN_MPA.COM, contendo:

$  set process/priv=syslck  
$  MPA_RUN = "$ " + F$PARSE("MPA-RUN.EXE", "[direxec]",,,)  
$  MPA_RUN ’P1’ ’P2’ ’P3’ ’P4’ ’P5’ ’P6’ ’P7’ ’P8’ ’P9’  
$  exit

Seguindo com o exemplo dado, a execução pode ser interativa

$  @RUN_MPA --host 10.10.228.253 --port 9091

Ou RUN_MPA.COM pode ser submetido a uma fila de execução em batch (em “lote”) com o comando

$  submit/log=[] RUN_MPA.COM /param=("--host 10.10.228.253", "--port 9091")

Quando a fila executar o arquivo RUN_MPA.COM, carregará o servidor com os parâmetros especificados.
