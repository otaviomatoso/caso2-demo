6.2.4 Atributos do Diagrama

Além de manipular os elementos de composição, a elaboração de um diagrama também consiste na definição de alguns atributos do diagrama. Esses atributos são definidos através do painel contextual quando nenhum elemento do diagrama está selecionado.


Figura 6.7: Painel de configuração de diagrama

O painel contextual permite a definição dos seguintes atributos de um diagrama:

Nome do diagrama
Nome pelo qual o diagrama será referenciado dentro do projeto.
Tipo do diagrama
Um diagrama pode ser do tipo aplicação ou do tipo função. Um diagrama-aplicação só possui uma única instância de execução no servidor (apesar de poder ter várias sub-tarefas concorrentes), enquanto um diagrama-função pode ter várias execuções concorrentes. Diagramas-aplicação podem ser iniciados ou encerrados, enquanto diagramas-função podem ser chamados, assim como funções, e receber parâmetros e/ou retornar resultados.
Diagrama de Interrupção
O usuário é capaz de determinar um outro diagrama a ser executado após a interrupção do diagrama. Interrupção essa que pode ser ocasionada tanto manualmente pelo operador, através da aba Execução (veja seção 8), quanto programaticamente, através de um comando que interrompa o diagrama, como explicado na subseção sobre operações sobre diagramas (seção 6.3.2). Todos os tipos de diagrama podem ter um diagrama de interrupção, mas somente diagramas-função que não recebam parâmetros podem ser definidos como diagramas de interrupção de outros diagramas.
Função de Erro
A função que deve ser chamada caso algum erro ocorra durante a execução do diagrama. Apenas as funções auxiliares que recebem um único parâmetro do tipo texto podem ser definidas como função de erro. Se houver um erro na execução do diagrama, essa função será chamada recebendo como parâmetro uma mensagem textual descrevendo o erro do diagrama.
Variáveis Globais
As variáveis globais do diagrama podem ser definidas através de uma tabela no painel contextual, como pode ser visto também na figura 6.7. Acima da tabela, há botões para adicionar, remover ou reposicionar a linha com foco da tabela. Uma nova linha aparece sempre no final da tabela para permitir a entrada de dados para uma nova variável global. Os botões com setas são utilizados para mudar a ordem das linhas das tabelas. O botão com a seta para cima move a linha selecionada uma posição para cima. Já o botão com a seta para baixo move a linha selecionada uma posição para baixo. Cada linha representa uma variável global e apresenta em suas colunas os dados que devem ser definidos para as mesmas (veja a seção 6.3.2 para mais detalhes sobre as expressões que podem ser usadas na definição de globais):
Variável
Deve ser um identificador válido (letras, _ ou algarismos, desde que não começe com um algarismo).
Tipo
Ao editar essa célula é apresentado um diálogo para seleção do tipo. Este diálogo também apresenta uma caixa de seleção a ser utilizada para indicar se o valor da variável é uma lista de valores ou não.
Valor
O preenchimento desse campo varia de acordo com o tipo selecionado. Quando o tipo for um tipo básico, o valor pode ser o próprio valor ou uma expressão, como 24*60 ou x+2 (fazendo referência a globais definidas em linhas prévias). Quando o tipo for uma classe de ponto ou equipamento, o campo listará as instâncias configuradas na classe em questão. Se o tipo for indicado como uma lista de valores, deve-se utilizar chaves e separar os valores ou tags de instâncias da lista usando vírgula ou ponto-e-vírgula, como em {1,2,3} ou {LIC-01,LIC-02,LIC-03}. Uma lista vazia pode ser criada com a expressão {}.
Descr.
Texto livre que pode ser usado para descrever o uso da variável no diagrama.
Modo
Só pode ser alterado no caso de fluxos-função. O modo indica se a global é um parâmetro ou valor de retorno do diagrama, ou mesmo se funciona como ambos.

Variáveis globais são compartilhadas por todas as linhas de execução de um diagrama, mas não são compartilhadas por diferentes instâncias de execução do mesmo diagrama (no caso de diagramas-função).
