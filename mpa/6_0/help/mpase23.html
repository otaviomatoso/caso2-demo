8.2 Aba de Monitoração

Um recurso útil para o desenvolvedor de aplicações MPA é o monitoramento de execução dos diagramas de controle. Assim como a depuração visual, o monitoramento permite verificar a corretude das manobras descritas através dos diagramas de controle. A aba Monitoração exibe as mensagens de monitoramento de execução de diagramas que são recebidas do servidor de execução. Esta tabela tem quatro colunas, que informam o horário de recebimento da mensagem, o nome do diagrama, a linha de execução que originou a mensagem e um texto descritivo. A IDE permite o monitoramento de diagramas em diferentes níveis de detalhe, o que é feito através da coluna Monitoramento da tabela da aba Execução no painel inferior. Há cinco opções de monitoramento, como descritas a seguir:

Sem Monitoramento
Esta opção desabilita toda forma de monitoramento de execução do diagrama, incluindo possíveis erros da execução.
Apenas Erros
Esta opção habilita o monitoramento de situações de erro na execução do diagrama, como uma soma de dois valores não numéricos ou uma divisão por zero. Todas as mensagens de monitoramento de erros são exibidas com o fundo amarelo e o texto da mensagem é apresentado na cor vermelha.
Acompanhamento
Esta opção habilita o recebimento das mensagens de acompanhamento que são explicitamente definidas na construção dos diagramas. O texto das mensagens de acompanhamento é exibido na cor azul.
Elementos do Diagrama
Esta é a opção de maior nível de detalhe, pois ela permite acompanhar passo a passo a execução dos elementos do diagrama de controle. Para cada elemento que compõe os diagramas é gerada uma mensagem diferente que indica o que foi feito pelo elemento. As mensagens geradas seguem uma estrutura intuitiva e similar à linguagem de programação Lua, sobre a qual o MPA está construído. Todas as mensagens de monitoramento dos elementos do diagrama são exibidas com o fundo azul claro. As mensagens com o texto na cor cinza indicam que o elemento já foi concluído. As mensagens com o texto na cor verde indicam que o elemento ainda está em execução. As mensagens com o texto na cor amarela indicam que o elemento atingiu um ponto de parada. Note, que quando um diagrama contém mais de uma linha de execução concorrente haverá mais de um elemento em execução para aquele diagrama. Além disso, comandos de iteração sobre equipamentos permanecem em execução (com o texto na cor verde) até o final da última iteração.

Assim como a tabela de diagramas da aba Execução, a lista de mensagens também botões laterais que permitem executar ações sobre a lista de mensagens, assim como definir o seu comportamento. A seguir é feita uma descrição dos botões da lista de mensagens.

Botão de Excluir Mensagens O primeiro botão é utilizado para limpar a caixa de mensagens recebidas do servidor. Após a exclusão dessas mensagens não é possível adicioná-las novamente ao buffer de mensagens, entretanto, é possível definir um arquivo onde as mensagens excluídas são armazenadas. Neste caso, as mensagens removidas da caixa de mensagem são armazenadas neste arquivo.

Botão de Rolagem Automática O segundo botão permite ativar/desativar a rolagem automática da lista de mensagens, de forma que as últimas mensagens recebidas estejam sempre visíveis.. Caso esse botão esteja pressionado, a lista de mensagens é rolada automaticamente de forma a sempre permitir a visualização da última mensagem recebida. Nota-se que caso o rolamento automático esteja ativado, mesmo que o usuário utilize a barra de rolagem para visualizar as mensagens anteriores, ao receber uma nova mensagem ela será adicionada à lista e esta será rolada de forma a exibir as últimas mensagens. Além disso, caso o buffer de mensagens esteja cheio, as mensagens serão deslocadas dentro do buffer, dando a impressão de que a lista de mensagem esteja com o rolamento automático.

Botão de Salvamento de Mensagens Excluídas O botão mais abaixo permite ativar/desativar o salvamento automático das mensagens excluídas da lista de mensagens para o arquivo de armazenamento definido através do comando Editar → Preferências. Caso esse botão esteja pressionado, o salvamento automático é ativado. Neste caso, todas as mensagens que forem retiradas do buffer sejam armazenadas num arquivo de mensagens, inclusive quando o buffer é explicitamente esvaziado.
