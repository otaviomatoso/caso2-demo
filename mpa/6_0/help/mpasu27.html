6.3.5 Executor

O elemento de composição Executor é representado graficamente por um círculo, como ilustrado na figura 6.24, e permite a criação de linhas de execução concorrentes. Sempre que o fluxo atinge um objeto Executor uma nova linha de execução independente é criada para executar cada um dos ramos do fluxo que saem deste elemento. Após a criação das linhas de execução concorrentes, a execução do ramo do fluxo que atingiu o Executor é suspensa e pode ser recuperada após o término da execução das linhas, como será visto na seção 6.3.6. Na figura 6.24, o elemento executor cria três linhas de execução concorrentes, sendo que cada uma executa um dos ramos do fluxo que saem dele.


Figura 6.24: O elemento de composição Executor.

O Executor é comumente utilizado após um Iterador quando deseja-se que um mesmo conjunto de comandos seja executado concorrentemente para diferentes instâncias de uma mesma classe. Um exemplo similar ao apresentado na seção 6.3.4 é exemplificado na figura 6.25.


Figura 6.25: Exemplo de uso do elemento executor.

Há diferenças importantes entre os dois casos:

  • Os elementos de composição a partir do Executor executam concorrentemente. Não há nenhuma espera para que os comandos começem a atuar sobre cada um dos tanques, uma vez que ao atingir o elemento Executor uma nova linha de execução independente é criada e aquele laço de iteração é encerrado, fazendo com que o Iterador avançe para a próxima iteração.
  • A sequência do fluxo (i.e. a partir da Ligação F) é independente da execução do ramo do fluxo que segue o Executor, portanto não há sincronismo entre esses ramos. O elemento de composição Tanques em fechamento é processado normalmente enquanto os comandos após o Executor são executados concorrentemente. Vale ressaltar que não há garantias quanto à ordem de execução das linhas concorrentes criadas pelo objeto Executor.
  • Os ramos originados em um objeto Executor formam ramos de fluxo independentes, portanto não podem se ligar com outros ramos do fluxo. Se houver alguma ligação entre ramos independentes, a verificação do fluxo irá informar o usuário dessa construção ilegal.
  • Cada linha de execução concorrente tem um contexto independente das demais, para onde as variáveis locais pré-existentes são copiadas, eliminando o compartilhamento. Além disso, é possível criar novas variáveis nesses contextos. Isso é útil para impedir que o processamento feito por uma linha de execução interfira na execução das demais, mesmo que elas executem o mesmo ramo do fluxo. Em contrapartida, as variáveis globais do fluxo continuam sendo compartilhadas entre as linhas de execução.
