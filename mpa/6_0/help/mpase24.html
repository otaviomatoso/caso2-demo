10.1 Arquivo de Parâmetros

Todas as configurações do servidor de execução podem ser feitas através de parâmetros de linha de comandos, como será descrito no capítulo 11. Alternativamente, é possível definir um arquivo de configuração contendo os parâmetros adequados para configurar o servidor de execução. É recomendável que sempre haja um arquivo com parâmetros com a configuração padrão do servidor, visto que todos esses parâmetros podem ser redefinidos com os valores dos parâmetros passados pela linha de comandos. A figura 10.1 ilustra o conteúdo de um arquivo de parâmetros de configuração do servidor de execução.


Figura 10.1: Arquivo de parâmetros de configuração.
 
-- Interface de Rede 
HOST      = ”10.20.30.40”; 
PORT      = 8080; 
 
-- Seguranca 
PASSWORD  = ”senhadeacesso”; 
 
-- Pre-Configuracao 
PRECONFIG = ”C:\MPA\Plantas\P31\preconfig.mpa”; 
 
-- Ponte de Comunicacao 
BRIDGE    = ”opc(Matrikon.OPC.Simulation,192.168.0.2)”; 
 
-- Configuracao Inicial 
LOAD      = ”C:\MPA\Plantas\P31\config.lua”; 
EXECUTE   = ”Gerencia_Modulos”; 
 
-- Arquivos de Log 
LOGFILE   = ”mpa.log”; 
LOGFLAGS  = ”m,l,p,t”;

É possível ter vários arquivos de parâmetros num único ambiente de automação. Entretanto apenas um pode ser utilizado para configurar cada servidor. O arquivo de parâmetros a ser utilizado pode ser especificado através da variável de ambiente MPA_CONFIG ou na linha de comando através do parâmetro –settings, como no exemplo abaixo:

mpa-run.exe --settings C:\MPA\main.cfg

Caso arquivo de parâmetros de configuração não seja definido usando nenhuma das formas descritas anteriormente, então o arquivo main.cfg é utilizado, caso esteja presente no diretório corrente do servidor.
